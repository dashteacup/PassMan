Manual Test Plan

Make a new file via the tool bar

  1. Open the app. You should see:
  2. Click the New file button on the toolbar
  3. A new empty text area should open up.
  4. Enter the text "Hello world!"
  5. Click the Save As button on the toolbar
  6. The save as dialog box should load. Enter the file name "newname.pman" and hit the save button. (This test assumes there is no file already named "newname.pman").
  7. The enter password dialog should appear. Enter "monkey" in the field and click Ok
  8. Now close the app.
  9. Restart the app and click the open file button on the toolbar
  10. The open file dialog box should appear. Select the "newname.pman" file and click open.
  11. The enter password dialog will pop up again. Enter the password "monkey" one more time.
  12. The file we created earlier should now show up in the text area again.

Make a new file using the file menu

  1. Repeat the steps of the previous test using the file menu instead of the toolbar buttons
  2. Make the file name "filenewname.pman" to avoid a clash.
  3. Make the password "12345" to prove we haven't hardcoded the pw field like I did in earlier versions.

Make a new file when the current file has unsaved changes

  1. Open the app.
  2. Using the file menu, select the open file menu item
  3. Select the file "midsize.pman"
  4. Enter the password "mommy" in the password dialog box
  5. Open the file
  6. Type the text "This isn't going to be saved." into the bottom of the text field
  7. Click the new file button on the toolbar
  8. The "You have unsaved changes" dialog should appear.
  9. Click No.
  10. A new empty file should appear.
  11. Click the open toolbar button
  12. Reopen "midsize.pman".
  13. Enter the password "mommy" in the password dialog box again.
  14. Verify that the changes we didn't save last time are no longer present.
